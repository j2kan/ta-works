Step Process Description
1 Password Change If you are a new AC, the password needs to be changed to maintain a secure system. In addition, use your discretion to determine when the password should be changed.
2 Upload Courses For each term, a csv file needs to be uploaded that identifies important information regarding each of the courses that will be offered. This task must be completed prior to the TA Application Form becoming live.
3 Upload Front Matter The TA Application Form has a Front Matter section. This section identifies important information that graduate students need to be aware of prior to applying. Should this information change, you are able to change the content through this process.
4 TA Application When the upcoming terms courses have been added to the system, the AC must make the TA Application Form 'live'. This allows graduate students to apply for a TA position. The AC must then close the form when the application period has ended. The AC may also preview the form here.
5 Assign Number of TAs The AC is responsible for determining the number of TA's that are required for each course in the upcoming term. These values may be changed while the TA Application Form is live, but should not be changed when applications are not longer accepted.
6 Ranking Status Once the application period has ended, the AC is responsible for notifying Instructors that they need to rank appplicants. System generated emails can be sent out to all Instructors here. This step will also provide a summary of which Instructors have ranked their applicants. The AC also has the ability to override Instructor preference for each course in this step.
7 Export Results Before running the matching algorithm, the AC must export the number of TAs required for each course as well as student and Instructor preferences. This must be completed for the algorithm to run. The TA Application Form should be 'closed' before this step happens.
